# Deep Research

体系的な多角リサーチスキル。単一の検索では不十分な場合に、複数の角度・深度からWeb調査を行う。

## トリガー

- 「〜について調べて」「リサーチして」「競合分析して」「SEO調査」
- コンテンツ作成前の事前調査が必要な場面

## 原則

**一般知識だけでコンテンツを生成しない。** 出力の質はリサーチの質と量に直結する。1回の検索では絶対に不十分。

## リサーチ手法

### Phase 1: 広域探索

まず全体像を把握する:

1. **初期サーベイ**: メイントピックを検索して全体的な文脈を理解
2. **次元の特定**: 初期結果からキーとなるサブトピック・テーマ・角度を特定
3. **領域マッピング**: 異なる視点・ステークホルダー・立場を記録

```
例: 「赤ちゃん 名付け トレンド」
初期検索:
  - "赤ちゃん 名前 人気ランキング 2026"
  - "姓名判断 画数 意味"
  - "名付け 親 悩み アンケート"

特定された次元:
  - 人気名前ランキング（男女別・年別推移）
  - 姓名判断の流派・計算方法の違い
  - 名付けで重視するポイント（音・意味・画数・字面）
  - 法的制約（人名用漢字の範囲）
  - 文化的背景（季節・自然・古典由来）
```

### Phase 2: 深掘り

特定された各次元について精密な調査を行う:

1. **具体的クエリ**: 各サブトピックに対して正確なキーワードで検索
2. **複数の表現**: 異なるキーワードの組み合わせ・言い回しを試す
3. **全文取得**: 重要なソースは WebFetch で全文を読む（スニペットだけで判断しない）
4. **参照追跡**: ソースが言及する他の重要リソースも検索

### Phase 3: 統合

1. **情報の整理**: ソース別にファクトをまとめる
2. **矛盾の解決**: 情報源間の矛盾を特定し、信頼性の高いソースを優先
3. **ギャップ分析**: まだ不足している情報を特定し、追加調査

## 検索のベストプラクティス

- **1トピックにつき最低3-5回検索**（異なる角度から）
- **日本語 + 英語の両方**で検索して情報量を最大化
- **最新情報を優先**: 年号を含めたクエリ（例: "2026"）
- **権威あるソース優先**: 公的機関、学術論文、大手メディア


## SEO 調査への応用

名前診断サービスの SEO 調査では以下を重点的に調べる:
- 検索ボリューム（Google Trends, キーワードプランナー）
- 競合サイトの見出し構造・コンテンツ量
- 上位表示サイトの共通パターン
- ユーザーの検索意図（情報探索型 vs ツール利用型）
